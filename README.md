# 城乡建设政策知识服务项目管理

## 📋 项目概述

本项目是**城乡建设政策知识服务**的核心工作区，旨在将政策洞察转化为可执行的项目方案，解锁下游设计和施工合同。

### 核心职责

1. **政策学习与知识库构建**
   - 自动爬取、解析和归档中央和地方政策文档
   - 将PDF和图像通过OCR转换为Markdown
   - 构建可搜索的知识图谱，链接法律、法规、资金窗口和项目类型
   - 维护版本化历史并归档过时政策

2. **项目谋划支持**
   - 接收项目概念后，匹配所有适用的政策激励
   - 推荐最优资金组合：专项债、超长期国债、中央预算资金、PPP、绿色金融
   - 起草合规路线图：审批序列、文档清单、里程碑时间表
   - 量化政策风险并提出缓解条款
   - 交付即提交包：可行性研究、资金申请、风险登记册

3. **咨询响应**
   - 将查询分类为：资金、审批、法律、技术标准、案例先例
   - 运用第一性原理思维揭示根本约束（现金流稳定性、土地权属、环境阈值）
   - 应用系统思维映射利益相关者网络（发改委、财政部、住建部、银行、地方政府）
   - 提供分层答案：1段执行摘要、2页简报、带引用的完整报告
   - 附带代码片段或Python笔记本进行数据计算

4. **深度政策分析**
   - 对每个高影响力政策，生成10页简报：背景、文本分析、链接图、影响预测、实施路径、案例研究、风险预警
   - 运用类比推理：与先前的周期（如2015年补贴转变）和跨部门法律（如《环境保护法》对施工许可的影响）进行比较
   - 嵌入交互式图表（Plotly），显示资金配额、区域分配趋势、审批率
   - 以可操作的洞察总结：谁赢、谁输、早期机会

5. **演示材料生成**
   - 将分析转化为15页中文幻灯片：问题、政策窗口、解决方案路线图、融资结构、风险矩阵、下一步
   - 自动生成带谈话要点和预期问答的演讲者备注
   - 导出为PPTX和PDF；确保与政府模板的视觉一致性

## 🚀 快速开始

### 环境设置

1. **激活Python虚拟环境**
   ```bash
   cd F:\AIlm\000项目管理
   .venv\Scripts\activate
   ```

2. **检查环境**
   ```bash
   python 02_工具和脚本\00_环境管理\check_resources.py
   python 02_工具和脚本\00_环境管理\check_python_env.py
   ```

3. **更新Skills**
   ```bash
   02_工具和脚本\01_Skills管理\auto_update_skills.bat
   ```

### 工作流程

1. **政策学习**
   - 查看 `01_政策知识库/00_政策索引/` 了解最新政策
   - 运行政策爬虫脚本更新知识库
   - 归档过时政策到 `01_政策知识库/05_政策归档/`

2. **项目谋划**
   - 使用 `00_项目管理核心/模板/` 中的模板
   - 参考 `00_项目管理核心/工作流程/` 中的流程
   - 输出文档保存到 `03_文档和报告/` 和 `05_输出和成果/`

3. **咨询响应**
   - 查询政策知识库
   - 生成分层答案
   - 保存到 `03_文档和报告/01_政策解读/`

4. **深度分析**
   - 使用数据分析脚本
   - 生成交互式图表
   - 输出报告到 `03_文档和报告/`

## 🎯 命名规范

### 文件夹命名
- 使用数字前缀排序：`00_`, `01_`, `02_`...
- 使用中文描述：`政策知识库`、`工具和脚本`
- 避免特殊字符

- 日期格式：`YYYY-MM-DD`（例如：`2025-02-28`）
