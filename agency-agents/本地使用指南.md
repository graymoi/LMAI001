## 概述

### 什么是Agency-Agents？

Agency-Agents是一个包含**131个专业化AI智能体**的完整集合，每个智能体都有：
- **专业化身份**：深度领域专业知识
- **个性特征**：独特的沟通风格和工作方式
- **工作流程**：经过验证的交付流程
- **质量标准**：可衡量的成功指标

### 核心特性

- **131个专业智能体**：覆盖工程、设计、营销、销售、项目管理等12个领域
- **NEXUS协调框架**：支持Full、Sprint、Micro三种工作模式
- **质量门控机制**：Dev↔QA循环确保交付质量
- **智能体协调**：无缝的上下文传递和交接
- **灵活集成**：支持Claude Code、Cursor、Aider等多种工具

## 快速开始

### 1. 激活单个智能体

#### 使用Python脚本

```python
from trae.skills.agency_agents_adapter import AgencyAgentsAdapter

# 创建适配器实例
adapter = AgencyAgentsAdapter()

# 激活前端开发智能体
result = adapter.activate_agent(
    agent_name="Frontend Developer",
    task="Build a React component for user authentication",
    context={
        "framework": "React",
        "styling": "Tailwind CSS",
        "requirements": "Login form with email and password"
    }
)

print(f"状态: {result.status}")
print(f"输出: {result.output}")
```

## NEXUS协调框架

### NEXUS模式对比

| 模式 | 适用场景 | 智能体数量 | 时间周期 | 阶段数量 |
|------|---------|-----------|---------|-----------|
| **NEXUS-Full** | 完整产品开发 | 所有智能体 | 12-24周 | 7个阶段 |
| **NEXUS-Sprint** | 功能/MVP开发 | 15-25个智能体 | 2-6周 | 5个阶段 |
| **NEXUS-Micro** | 特定任务 | 5-10个智能体 | 1-5天 | 1个阶段 |

### NEXUS-Full阶段详解

#### Phase 0: Discovery（发现）
**智能体：** Trend Researcher, Feedback Synthesizer, UX Researcher, Analytics Reporter, Legal Compliance Checker, Tool Evaluator

**目标：**
- 市场趋势分析
- 用户反馈收集
- 用户体验研究
- 数据分析
- 法律合规检查
- 工具评估

#### Phase 1: Strategy（策略）
**智能体：** Studio Producer, Senior Project Manager, Sprint Prioritizer, UX Architect, Brand Guardian, Backend Architect, Finance Tracker

**目标：**
- 项目策略制定
- 项目规划
- 冲刺优先级排序
- UX架构设计
- 品牌指导
- 后端架构设计
- 财务跟踪

#### Phase 2: Foundation（基础）
**智能体：** DevOps Automator, Frontend Developer, Backend Architect, UX Architect, Infrastructure Maintainer

**目标：**
- DevOps设置
- 前端基础
- 后端基础
- UX实现
- 基础设施搭建

#### Phase 3: Build（构建）
**智能体：** Frontend Developer, Backend Architect, DevOps Automator
**QA智能体：** Evidence Collector, API Tester, Performance Benchmarker

**目标：**
- 功能开发
- Dev↔QA循环
- 质量验证

#### Phase 4: Harden（加固）
**智能体：** Reality Checker, Performance Benchmarker, API Tester, Legal Compliance Checker

**目标：**
- 现实性检查
- 性能优化
- API测试
- 合规验证

#### Phase 5: Launch（发布）
**智能体：** Growth Hacker, Content Creator, DevOps Automator

**目标：**
- 增长策略
- 内容创作
- 部署上线

#### Phase 6: Operate（运营）
**智能体：** Analytics Reporter, Infrastructure Maintainer, Support Responder

**目标：**
- 数据分析
- 基础设施维护
- 用户支持

### Dev↔QA循环

#### 循环流程

1. **开发阶段**：开发智能体实现功能
2. **QA验证**：Evidence Collector验证实现
3. **反馈循环**：
   - ✅ PASS：进入下一个任务
   - ❌ FAIL：返回开发修复（最多3次）
4. **升级机制**：3次失败后升级处理

#### 质量门控

每个阶段必须通过质量门控才能进入下一阶段：

```python
def quality_gate(phase_result):
    """
    质量门控检查
    
    检查项：
    - 所有智能体都成功完成
    - 交付物符合要求
    - QA验证通过
    - 无阻塞性问题
    """
    for agent_result in phase_result.values():
        if agent_result.status != "SUCCESS":
            return False
    
    return True
```

### 4. 与现有智能体集成

#### 更新项目管理智能体

```python
from trae.skills.agency_agents_adapter import AgencyAgentsAdapter, NexusOrchestrator, NexusMode

class ProjectManagerAgent:
    """项目管理智能体（增强版）"""
    
    def __init__(self):
        self.agency_adapter = AgencyAgentsAdapter()
    
    def plan_project(self, project_spec: str) -> dict:
        """使用Senior Project Manager智能体规划项目"""
        result = self.agency_adapter.activate_agent(
            agent_name="Senior Project Manager",
            task=f"Create project plan from specification",
            context={"spec": project_spec}
        )
        return result
    
    def execute_project(self, project_name: str, mode: str = "sprint") -> bool:
        """使用NEXUS框架执行项目"""
        nexus_mode = NexusMode.SPRINT if mode == "sprint" else NexusMode.FULL
        
        orchestrator = NexusOrchestrator(
            adapter=self.agency_adapter,
            mode=nexus_mode,
            project_name=project_name,
            spec_path=f"F:/AIlm/000项目管理/标书编写/{project_name}.md"
        )
        
        return orchestrator.execute_pipeline()
```

## 常见问题

### Q1: 如何选择合适的智能体？

**A:** 使用搜索和推荐功能：

```python
# 搜索智能体
results = adapter.search_agents("react frontend")

# 推荐智能体
recommendations = adapter.recommend_agents("Build a React application")
```

### Q2: NEXUS模式如何选择？

**A:** 根据项目规模选择：

- **NEXUS-Micro**：单个任务、Bug修复、小功能（1-5天）
- **NEXUS-Sprint**：MVP、功能开发（2-6周）
- **NEXUS-Full**：完整产品、企业级应用（12-24周）

### Q3: 质量门控失败怎么办？

**A:** 检查以下几点：

1. 所有智能体是否都成功完成
2. 交付物是否符合要求
3. QA验证是否通过
4. 是否有阻塞性问题

如果问题持续，可以：
- 调整任务分解
- 增加资源投入
- 重新评估需求
