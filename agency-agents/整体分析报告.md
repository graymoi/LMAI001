### 2. 核心功能实现

#### 2.1 智能体管理

**功能：**
- 智能体加载和缓存
- 智能体搜索（基于关键词）
- 智能体推荐（基于任务描述）
- 智能体激活（生成提示模板）

**性能指标：**
- 智能体加载速度：< 100ms
- 智能体搜索速度：< 50ms
- 智能体推荐速度：< 100ms

#### 2.2 NEXUS协调框架

**功能：**
- 三种工作模式（Full/Sprint/Micro）
- 7阶段管道（Full模式）
- 5阶段管道（Sprint模式）
- Dev↔QA循环
- 质量门控机制

**阶段对比：**

| 模式 | 阶段数 | 智能体数 | 时间周期 | 适用场景 |
|------|---------|-----------|---------|---------|
| **NEXUS-Full** | 7 | 所有 | 12-24周 | 完整产品开发 |
| **NEXUS-Sprint** | 5 | 15-25 | 2-6周 | MVP/功能开发 |
| **NEXUS-Micro** | 1 | 5-10 | 1-5天 | 特定任务 |

#### 2.3 智能体协调

**功能：**
- 多智能体激活
- 智能体交接（上下文传递）
- 共享上下文管理
- 交接历史记录

**交接模板：**
```markdown
# Handoff: [From Agent] → [To Agent]

## Context
- 项目: [Project Name]
- 阶段: [Current Phase]
- 规格: [Spec Path]

## Deliverables from [From Agent]
- [Deliverable 1]: [Description]
- [Deliverable 2]: [Description]

## Instructions for [To Agent]
Please review deliverables and continue workflow...

## Quality Checklist
- [ ] All deliverables reviewed
- [ ] Context understood
- [ ] Next steps identified
```

## 📖 使用指南

### 快速开始

#### 1. 安装Agency-Agents

```bash
# 克隆仓库
git clone https://github.com/msitarzewski/agency-agents.git

# 验证安装
find agency-agents -name "*.md" | wc -l
# 应该显示131个智能体文件
```

#### 2. 使用Python脚本

```python
from trae.skills.agency_agents_adapter import AgencyAgentsAdapter

# 创建适配器
adapter = AgencyAgentsAdapter()

# 搜索智能体
results = adapter.search_agents("react frontend")
for result in results:
    print(f"{result['name']} (分数: {result['score']})")

# 激活智能体
result = adapter.activate_agent(
    agent_name="Frontend Developer",
    task="Build a React component for user authentication",
    context={
        "framework": "React",
        "styling": "Tailwind CSS"
    }
)
```

#### 3. 使用NEXUS框架

```python
from trae.skills.agency_agents_adapter import NexusOrchestrator, NexusMode

# 创建NEXUS协调器
orchestrator = NexusOrchestrator(
    adapter=adapter,
    mode=NexusMode.SPRINT,
    project_name="背街小巷诊断数字化管理平台",
    spec_path="F:/AIlm/000项目管理/标书编写/需求规格说明书.md"
)

# 执行管道
success = orchestrator.execute_pipeline()
```

## 💡 最佳实践

### 1. 智能体选择

**原则：**
- 根据任务类型选择合适的智能体
- 提供详细的上下文信息
- 使用搜索和推荐功能找到最佳匹配

**示例：**
```python
# ❌ 不好：过于泛化
adapter.activate_agent("Developer", "Build something")

# ✅ 好：具体明确
adapter.activate_agent(
    agent_name="Frontend Developer",
    task="Build a user authentication form with React and Tailwind CSS",
    context={
        "framework": "React",
        "styling": "Tailwind CSS",
        "features": ["email", "password", "remember_me"],
        "validation": "client-side and server-side"
    }
)
```

### 2. NEXUS模式选择

**决策树：**

```
开始
  │
  ├─ 是否是完整产品开发？
  │   ├─ 是 → NEXUS-Full（12-24周）
  │   └─ 否 → 继续
  │
  ├─ 是否是MVP或功能开发？
  │   ├─ 是 → NEXUS-Sprint（2-6周）
  │   └─ 否 → 继续
  │
  └─ 是否是单个任务或Bug修复？
      ├─ 是 → NEXUS-Micro（1-5天）
      └─ 否 → 重新评估
```

### 3. 质量门控

**原则：**
- 严格执行质量门控
- 不跳过QA验证
- 最多3次重试
- 失败后升级处理

**示例：**
```python
# 配置严格的质量门控
config = {
    "quality_gate_strict": True,
    "max_retries": 3,
    "qa_agents": [
        "Evidence Collector",
        "Reality Checker",
        "API Tester"
    ]
}
```

### 4. 上下文管理

**原则：**
- 提供充分的上下文
- 保持上下文的一致性
- 及时更新共享上下文
- 记录重要的决策

**示例：**
```python
coordinator = AgentCoordinator()

# 初始化上下文
coordinator.update_context("project", "背街小巷诊断数字化管理平台")
coordinator.update_context("phase", "Development")
coordinator.update_context("tech_stack", "React + Laravel")

# 在后续任务中使用
result = coordinator.activate_agent(
    agent_name="Frontend Developer",
    task="Build dashboard component"
    # 上下文自动传递
)
```
