# 标书内容工程技能

## 技能概述

本技能专门用于标书编写的内容工程，避免通用AI语调，保持专业、规范的文档风格。基于everything-claude-code的article-writing技能优化而来，针对中国政府采购标书编写场景定制。

---

## 核心功能

### 1. 专业语调控制

**功能描述：** 确保所有标书文档使用专业、规范的政府项目语调

**语调特征：**
- 正式、严谨、专业
- 避免口语化表达
- 使用标准政府采购术语
- 保持客观中立立场
- 遵循公文写作规范

**实施方法：**
```markdown
# 语调检查清单

- [ ] 避免使用"我觉得"、"我认为"等主观表达
- [ ] 使用"本系统"、"该方案"等客观表述
- [ ] 避免感叹号和问号
- [ ] 使用完整句子，避免省略
- [ ] 专业术语准确无误
- [ ] 数据和事实有据可查
```

### 2. 结构化内容生成

**功能描述：** 按照标准标书结构生成内容

**标准结构：**
```markdown
# 文档标题

## 一、项目概况
- 项目背景
- 建设目标
- 建设内容
- 建设规模

## 二、技术要求
- 总体要求
- 功能要求
- 性能要求
- 安全要求
- 验收标准

## 三、实施方案
- 技术方案
- 实施步骤
- 进度安排
- 人员配置
- 质量保障

## 四、服务承诺
- 服务内容
- 服务标准
- 响应时间
- 质量保证

## 五、报价说明
- 报价依据
- 费用明细
- 付款方式
```

### 4. 数据驱动内容

**功能描述：** 确保所有陈述都有数据支撑

**数据引用规范：**
```markdown

**数据来源：** [具体来源]

**数据类型：** [统计数据/调研数据/实验数据]

**示例：**
```

**功能描述：** 确保内容符合政府采购相关法规

**合规检查清单：**

## 工作流程

### 阶段1：需求分析

**输入：** 用户需求、参考文档、项目背景

**处理：**
1. 分析项目类型（技术方案/需求规格/实施计划）
2. 识别目标受众（评审专家/政府部门/技术团队）
3. 确定文档结构（标准结构/定制结构）
4. 收集相关资料（政策文件/技术标准/案例参考）

**输出：** 需求分析报告

### 阶段2：内容规划

**输入：** 需求分析报告

**处理：**
1. 制定内容大纲
2. 分配各章节字数
3. 确定关键术语
4. 规划数据引用
5. 设计图表位置

**输出：** 内容规划文档

### 阶段3：内容生成

**输入：** 内容规划文档

**处理：**
1. 生成各章节内容
2. 应用专业语调
3. 整合数据支撑
4. 插入图表说明
5. 确保逻辑连贯

**输出：** 初稿文档

### 阶段4：质量检查

**输入：** 初稿文档

**处理：**
1. 语调一致性检查
2. 术语准确性检查
3. 数据完整性检查
4. 合规性检查
5. 格式规范性检查

**输出：** 质量检查报告

### 阶段5：优化完善

**输入：** 质量检查报告

**处理：**
1. 修正发现的问题
2. 优化表达方式
3. 补充缺失内容
4. 提升可读性
5. 完善图表说明

**输出：** 最终文档

- **美观性：** 排版合理，视觉舒适

### 合规质量

- **法律合规：** 符合相关法律法规
- **技术合规：** 符合技术标准规范
- **财务合规：** 符合财务制度要求
- **程序合规：** 符合采购程序规定

```markdown
### 2.1 总体要求
系统应采用微服务架构，支持高并发访问，具备良好的扩展性和可维护性。
```
